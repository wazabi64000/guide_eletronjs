2. Qu'est-ce qu'une application Web ?

Une application Web est un logiciel accessible via un navigateur (Chrome, Firefox, Safari, Edge). Le navigateur télécharge des fichiers HTML, CSS et JavaScript et les exécute pour afficher une interface interactive.

3. Qu'est-ce qu'une application Desktop ?

Une application Desktop s'installe sur le système d'exploitation. Elle accède aux fichiers locaux, imprimantes, notifications système et bases de données locales.

Exemples Electron : VS Code, Slack, Discord, Notion Desktop, Obsidian.

4. Qu'est-ce qu'Electron ?

Electron est un framework open source (GitHub, 2013) qui combine Chromium (rendu UI) et Node.js (accès système) pour créer des applications Desktop en HTML, CSS et JavaScript.

┌─────────────────────────────────────────────────────────────────┐ │ APPLICATION ELECTRON │ │ ┌──────────────────────┐ ┌──────────────────────────────┐ │ │ │ MAIN PROCESS │ │ RENDERER PROCESS(ES) │ │ │ │ (Node.js) │◀──▶│ (Chromium) │ │ │ │ • Fenêtres │IPC │ • HTML / CSS / JS │ │ │ │ • Menus │ │ • Interface utilisateur │ │ │ │ • SQLite / fichiers │ │ │ │ │ └──────────────────────┘ └──────────────────────────────┘ │ │ └────────── PRELOAD (pont sécurisé) ──────────┘ │ └─────────────────────────────────────────────────────────────────┘

Main Process

  • Point d'entrée : main.js
  • Un seul par application — gère fenêtres, menus, cycle de vie
  • Accès complet Node.js et APIs Electron

Renderer Process

  • Un par fenêtre — affiche HTML/CSS/JS
  • Pas d'accès Node.js direct (sécurité)
  • Communique via IPC

Preload Script

  • Pont sécurisé via contextBridge.exposeInMainWorld()
  • Exécute avant le chargement de la page

7. Exemple simple — Hello Electron

// main.js — Main Process
import { app, BrowserWindow } from 'electron';
import path from 'node:path';

function createWindow() {
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      contextIsolation: true,   // Sécurité
      nodeIntegration: false, // Sécurité
    },
  });
  win.loadFile('index.html');
}

app.whenReady().then(createWindow);

10. Bonnes pratiques

  • Toujours contextIsolation: true et nodeIntegration: false
  • Exposer le minimum d'API via contextBridge
  • Séparer Main (système) et Renderer (UI)

11. Erreurs fréquentes

nodeIntegration: true partout → failles XSS critiques. Utilisez Preload + contextBridge.
